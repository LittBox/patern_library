# 前端需求复用提示模板

这份文档用于降低下次和 AI 协作时的沟通成本。适合用于文化展示站、非遗展示站、内容陈列型单页面项目，也可按需裁剪到其他前端项目中。

## 使用方式

- 如果是新项目，可以直接复制“完整提示模板”给 AI。
- 如果是已有项目，建议先补上“项目上下文补充项”，再粘贴模板。
- 如果只做局部改版，可以只复制对应模块，例如导航、Hero、资讯、加载动画。

## 项目上下文补充项

在把需求发给 AI 之前，优先补齐下面这些信息：

- 技术栈：`HTML / CSS / JS` 或 `React / Vue / Next.js`
- 部署方式：`GitHub Pages / Netlify / Vercel / 自建服务器`
- 页面结构：单页面或多页面
- 是否允许改项目结构
- 是否允许改 JS 逻辑
- 现有主题色
- 现有静态资源目录
- 哪些模块属于重点页面
- 哪些功能绝对不能被样式改动破坏

## 完整提示模板

```text
请在不破坏现有功能的前提下，对当前前端进行优化，并把改动控制在最小必要范围内。

项目背景：
- 这是一个以文化展示 / 非遗展示 / 内容陈列为主的前端项目。
- 当前项目重点是“信息展示气质、文化符号表达、加载体验、内容层级”。
- 优先复用现有资源、现有样式语言和现有 DOM 结构，不要无意义重构。

视觉要求：
- 整体风格要偏中国文化展陈感，克制、留白、轻层次，不要做成后台风或营销弹窗风。
- 优先使用“米白底 + 中国红强调”的结构。
- 大面积底色可使用 `#fcf5e2`。
- 强调色、标题色、主按钮色可使用 `#b1231c`。
- 禁止使用渐变色。
- 尽量减少其他杂色。

字体要求：
- 一级标题优先使用厚重、清晰的中文标题字体。
- 导航和文化性短标签可单独使用更有文化感的字体。
- 正文优先保证可读性，不要为了风格牺牲阅读体验。

导航要求：
- 导航栏优先使用双层结构：上层品牌与日期信息，下层主导航。
- logo 在左，日期/农历/节气在右。
- 下层目录靠右，不要机械居中。
- 移动端使用抽屉导航。

首页 Hero 要求：
- 左侧固定尺寸轮播图，右侧信息区，整体左右各占一半。
- 轮播图容器大小固定，图片允许裁剪填充，但布局不能跳动。
- 信息区内容左对齐。
- 装饰性文化纹样可作为背景氛围层，但不能变形、不能喧宾夺主。

标题装饰要求：
- 中式边框装饰只用于页面一级标题。
- 使用左上和右下两枚角标包裹标题。
- 不要旋转，不要额外给标题加背景块。
- 其他信息卡片标题不要套用这套装饰。

资讯/报道模块要求：
- 可分为“原创”和“转载”。
- 卡片优先展示：来源、日期、图片、标题、详情链接。
- 摘要不是必须，有时可以删除。
- 如果有重点媒体，优先置顶，例如央视网。
- 首页预览需要避免右侧重复出现同一系列内容。

图片加载要求：
- 不要在图片未完整到达时直接展示破碎占位态。
- 图片完整加载前先显示加载动画，加载完成后再一次性替换为真实图片。
- 加载动画优先使用 SVG 或 CSS。
- 动画应体现文化符号，线条简洁，有透明度层次。
- 需要支持 reduced motion 降级。

实现约束：
- 优先改样式，谨慎改 JS。
- 不要破坏现有接口请求、数据存储、上传、删除、AI 生成、详情弹窗、导航切换等功能。
- 不要随意改变事件绑定节点、id、data-* 标记。
- 静态资源路径必须先核对真实目录后再引用。
- 避免使用隐藏目录作为静态资源路径，例如 `img/.svg`。

输出要求：
- 直接完成代码修改，不只给方案。
- 改完后说明改了哪些模块。
- 如果发现某些需求和现有结构冲突，请先说明风险再处理。
- 最后补一个简短验收清单。
```

## 局部模块提示模板

### 1. 导航改版模板

```text
请只优化导航栏，不改动其他页面结构。

要求：
- 双层导航
- 左侧 logo + 站点名称
- 右侧日期 / 农历 / 节气
- 下层目录靠右
- 移动端使用抽屉导航
- 风格偏文化展示站
- 不要破坏原有切页逻辑
```

### 2. Hero 改版模板

```text
请只优化首页 Hero 区。

要求：
- 左右 1:1 布局
- 左侧固定尺寸轮播图
- 图片裁剪填充，容器不跳动
- 右侧信息区左对齐
- 加入轻量文化纹样背景
- 不要影响轮播逻辑和图片加载逻辑
```

### 3. 加载动画模板

```text
请优化图片加载体验。

要求：
- 图片完整加载前不要直接显示空白图框
- 使用 SVG/CSS 做文化符号加载动画
- 动画简洁、有层次、透明度有轻重
- 加载完成后再替换为真实图片
- 保留 reduced motion 降级
```

### 4. 资讯模块模板

```text
请新增一个资讯/报道模块，并复用现有样式语言。

要求：
- 分类为原创和转载
- 使用小卡片
- 卡片展示图片、标题、日期、来源、详情链接
- 详情直接跳转原平台
- 若存在重点媒体，优先置顶
- 首页预览不要重复堆叠同一系列内容
```

## 给 AI 的补充提醒

- 如果你修改了静态资源目录，请同步检查所有引用路径。
- 如果你新增了装饰元素，请确认没有覆盖交互热区。
- 如果你调整了轮播和图片加载逻辑，请确认图片不会半加载闪现。
- 如果你修改了卡片结构，请确认详情按钮、外链、删除、上架、改价等行为没有失效。
- 如果你新增了文化装饰，请优先克制，不要把页面做成“视觉堆砌”。
